   - Safe to expose in browser

✅ NEXT_PUBLIC_SUPABASE_ANON_KEY (REQUIRED)
   - Public anonymous key for Supabase
   - Used by client-side code
   - Safe to expose in browser

✅ SUPABASE_SERVICE_ROLE_KEY (REQUIRED)
   - Server-side admin key

⚡ EMERGENT_LLM_KEY (OPTIONAL)
   - For AI features: question generation, lesson content
   - Without this: users can still practice with manually created content
   - Get from Bolt/Emergent support

💳 STRIPE_API_KEY (OPTIONAL)

   - Without this: app works, but no payment processing
   - Get from https://dashboard.stripe.com/apikeys


TROUBLESHOOTING:
----------------

❌ Build still fails after adding variables?
   → Make sure you clicked "Redeploy" after adding them
   → Variables are NOT automatically applied to existing builds

❌ "Database not configured" error in production?
   → Double-check SUPABASE_SERVICE_ROLE_KEY is correct
   → Make sure you selected all three environments when adding the variable

❌ Users can't sign up?
   → Verify all 3 Supabase variables are added correctly
   → Check Supabase project is active (not paused)

❌ AI features not working?
   → EMERGENT_LLM_KEY needs to be added (optional feature)
